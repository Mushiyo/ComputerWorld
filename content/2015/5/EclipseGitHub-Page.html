title=使用Eclipse來建立GitHub Page date=2015-05-23 22:53 full_date=2015-05-23T22:53:00.000+08:00 type=post allowComment=true tags=eclipse, 筆記, GitHub status=published ~~~~~~ 本篇將介紹如何透過Eclipse中的EGit來建立GitHub Page。
GitHub Page是GitHub提供給其使用者的一個建立靜態網站的功能。
目前GitHub Page一共有二種，即User/Organization Page和Project Page。
底下只會提到Project Page，不過二種Page的建立步驟差異不大，相信在看完本篇後只要再看GitHub對這二種Page的說明後，即可以類似的方法建立User/Organization Page。
我對EGit的操作方式尚在學習中，若你發現有哪裡寫得不好或不對，請不吝留言指教。

0. 假設你有一個開好的GitHub repository，且你的Eclipse workspace中有一個寫好的網站Project，而這個Project已用了Git。

1. 在Eclipse的Project Explorer中，找到你要用來做為GitHub Page的那個網站Project，對Project名稱按下右鍵→Team→Push Branch...

如果這個Project已和GitHub連結，也就是之前有被push到GitHub上，接下來請看步驟2.1。若否，請看步驟2.2。

2.1  如果你的Project已和GitHub連結，也就是之前有被push到GitHub上，則畫面會類似下圖所示。檢查Remote那欄中的內容是否為你想建立GitHub Page的那個GitHub Repository。若否的話，按下"New Remote..."，跳至步驟2.2。若是的話，在Branch-name那欄打上"gh-pages"，注意這邊一定要打對，不然結果就會只是普通的branch。其他欄位的部份我也不知道那些是做什麼的，所以就不管它了（汗）。接著按下Next。

2.2 打開瀏覽器，接著如同要做git clone時的那樣，打開你想建立GitHub Page的那個GitHub Repository，在右欄處找到"clone URL"並複製起來。再回到Eclipse，在URI欄位貼上方才複製的內容。若內容正確無誤，Eclipse應該會自動幫你填好其他的欄位。按下Next。

3. 直接按下Finish

4. 按下OK

5. 登入GitHub，打開你方才push的那個repository，在按下右欄處的"Settings"。

6. 點開之後，往下捲動，找到一欄叫"GitHub Pages"的欄位。若你在那欄下方看到一行類似"Your site is published at http://username.github.io/projectName."的字樣，就表示成功了！點擊那行文字中的網址，就可以打開方才傳上去的網站。

7. 未來要再更新GitHub Page時，再照步驟1~4走一遍即可
